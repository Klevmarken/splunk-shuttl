Web Intelligence settings have been saved.

You will be able to view data beginning five minutes ago. If this is the first time you are running the Web Intelligence app, we recommend you perform two additional steps if (1) you want the dashboards to display historical data older than five minutes, and (2) you have multiple websites in your web environment.


1. Backfill your historical data.

The Web Intelligence app implements several levels of data summarization (5 minute, hourly, and daily) to enable searching large volumes of data efficiently. These summaries will populate automatically going forward but must be manually filled using our commandline backfill script for historical data. The script, called backfill_all.sh, is located in $SPLUNK_HOME/etc/apps/webintelligence/bin, and can be run using the command:

./backfill_all.sh <username> <password>

For detailed instructions on how to backfill your data, refer to the Web Intelligence Manual.


2. Defines sources for your data.

You can restrict searches to a specific website using a site selector. This site selector gets its values from a lookup table that you need to edit manually.

First, run the saved search called "Sourcenames Lookup" to populate the lookup table. You can run this search from the Search view:

| savedsearch "Sourcenames Lookup"

This search populates a csv file which lists multiple websites by their source. Once the search is complete, edit this lookup table in $SPLUNK_HOME/etc/apps/webintelligence/lookups/sourcenames.csv to replace the blank sourcename fields with a custom name for each source. You can map multiple sources to the same sourcename.
